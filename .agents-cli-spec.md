# Workstream Control Agent Spec

## Overview
Workstream Control Agent coordinates messy project context from meeting transcripts, Slack/email/Jira-style dumps, local repo state, and GitHub PR workflows. It maintains a persistent execution ledger that records sources, tasks, decisions, blockers, approvals, PR drafts, and memory updates.

The v1 capstone is prototype-first: mocked Jira/Slack/email adapters, real local ledger storage, real local repo inspection, and GitHub-ready PR draft generation. External writes are never executed without human approval.

## Example Use Cases
- Paste a meeting transcript and ask the agent to extract owners, deadlines, decisions, blockers, and follow-ups.
- Paste a Slack/email/freeform dump and ask the agent to reconcile duplicate work against existing tasks.
- Ask why a task exists and receive the source evidence, decision trail, and related PR draft.
- Ask for a Jira update, Slack reply, email reply, or GitHub PR body and receive a draft plus an approval request.
- Ask what is blocked and receive tasks with missing owners, conflicting deadlines, or unresolved dependencies.

## Tools Required
- `ingest_source`: store raw work context in the local ledger.
- `extract_work_items`: extract task, decision, and blocker candidates from a source.
- `reconcile_tasks`: merge or create canonical ledger tasks from candidates.
- `write_ledger_event`: append auditable event history.
- `update_project_memory`: update durable memory with source evidence.
- `inspect_repo`: inspect local Git status and recent commits.
- `draft_github_pr`: generate PR title/body/branch/commit message.
- `request_approval`: create approval records for external writes.
- Mock adapters for Jira, Slack, and email draft workflows.

## Constraints & Safety Rules
- The agent may update the local ledger automatically.
- The agent may draft external actions automatically.
- The agent must request approval before Slack posts, emails, Jira updates, GitHub PR creation, pushes, merges, or external status changes.
- The agent must not claim work is done, shipped, merged, or deployed without tool evidence.
- The agent must preserve source evidence for tasks, decisions, and status updates.
- The agent must redact secrets and sensitive identifiers before preparing external-facing drafts.

## Success Criteria
- Extracts tasks, owners, deadlines, decisions, and blockers from noisy inputs.
- Reconciles duplicate tasks across multiple sources.
- Maintains append-only events and inspectable memory files.
- Drafts Jira, Slack, email, and GitHub PR updates without executing external writes.
- Refuses unauthorized external write requests and creates approval records instead.
- Covers the 5-day course concepts: vibe coding/spec, tools/interoperability, skills, memory, HITL security, evals, and production path.
